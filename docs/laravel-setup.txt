In Laravel project root directory do
cd /home/sites/devx

sudo find -type f -exec chmod 644 {} \;
sudo find -type d -exec chmod 755 {} \;

chcon -Rt httpd_sys_content_rw_t storage /home/sites/devx
chcon -Rt httpd_sys_content_rw_t public

to check use ls -Z, you should see
unconfined_u:object_r:httpd_sys_content_t:s0 app
unconfined_u:object_r httpd_log_t:s0 logs
unconfined_u:object_r:httpd_sys_content_t:s0 framework

sudo chown apache.swang storage/*

selinux setup as following to external acess i.e., http://prod (71.59.72.103), http://devx not working (127.0.0.1)
setenforce 1
setsebool -P httpd_read_user_content 1
setsebool -P httpd_can_network_connect_db 1

If you want to allow httpd to read home directories, you must turn on the httpd_enable_homedirs boolean. Disabled by default.

setsebool -P httpd_enable_homedirs 1    -- this will make http://devx working !!!!

man httpd_selinux man page for details about the setsebool setup
